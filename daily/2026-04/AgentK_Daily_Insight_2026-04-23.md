## 4. Vercel 침해: OAuth 공격이 플랫폼 환경 변수의 숨은 위험 노출
- 링크: https://news.hada.io/topic?id=28768

[핵심 요약]
제3자 OAuth 토큰 탈취에서 시작된 침해가 Vercel 내부 시스템 접근과 일부 고객 프로젝트의 환경 변수 노출로 이어졌습니다. 특히 플랫폼 내부에서 열거 가능한 환경 변수가 하류 서비스 자격증명 노출로 이어질 수 있다는 점이 드러나며, 단일 OAuth 신뢰 관계가 공급망 전체 리스크로 증폭될 수 있음을 보여줬습니다.

[기술적 인사이트]
개발 플랫폼 보안의 핵심은 이제 코드 저장소나 CI만이 아니라 "환경 변수 저장 계층"입니다. 공급업체 OAuth 승인 목록 정리, 환경 변수 민감도 분리, 키 회전 자동화, 외부 유출 알림 연동을 별도 통제로 운영하지 않으면 SaaS 하나의 침해가 다수 서비스 침해로 번질 수 있습니다.

## 7. Garry Tan의 "Skillify" — AI 에이전트의 실패를 영구적 구조 수정으로 바꾸는 방법론
- 링크: https://news.hada.io/topic?id=28777

[핵심 요약]
Garry Tan이 에이전트의 실패를 일회성 프롬프트 수정으로 덮지 말고, 스킬 문서·결정론적 스크립트·자동화 테스트로 영구 수정하자는 "Skillify" 방법론을 제안했습니다. 실패를 회귀 테스트가 붙은 구조물로 전환하자는 소프트웨어 엔지니어링식 접근입니다.

[기술적 인사이트]

## 8. Anthropic - OpenClaw : 다시 사용 가능해짐
- 링크: https://news.hada.io/topic?id=28761

[핵심 요약]
Anthropic 계열 Claude CLI 재사용과 OpenClaw 스타일 통합이 다시 허용됐다는 소식입니다. API 키 기반 운영, adaptive thinking, prompt caching, 1M context 설정 등 실무 운영 옵션이 함께 정리되며, 모델 사용성보다 운영 세부설정이 더 중요한 국면임을 보여줍니다.
