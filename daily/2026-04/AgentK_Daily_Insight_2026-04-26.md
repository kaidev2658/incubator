## 22. 여러 GitHub 서비스 장애 사고

[핵심 요약]
Webhooks, Actions, Copilot 등 여러 GitHub 서비스에서 동시다발적 가용성 저하가 발생했습니다. 개발 파이프라인 핵심 서비스가 한 번에 흔들린 사례입니다.

[기술적 인사이트]
현대 개발 조직의 공급망 리스크는 코드 저장소 자체보다도 그 위에 얹힌 자동화 서비스 의존성에서 커집니다. CI/CD, 코드 리뷰, AI 보조도구가 같은 플랫폼에 묶여 있다면 장애는 곧바로 전달 속도 저하로 이어집니다.
