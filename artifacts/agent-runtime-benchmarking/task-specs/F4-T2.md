# F4-T2 — Recovery from Routine Command Failure

## Family
Family 4. Tool-use and execution loop

## Goal
Recover from a routine command failure and complete the task without entering a retry loop.

## Benchmark intent
This variant tests runtime resilience under ordinary execution friction rather than perfect command conditions.

## Expected behavior
- detect the command failure
- choose a sensible next action
- retry only when justified
- complete or escalate cleanly

## Success criteria
- failure detected correctly
- retry strategy is sensible
- no blind repeated loop
- final report explains what happened

## Failure examples
- same command retried repeatedly without diagnosis
- failure ignored
- incorrect recovery path
- false success claim after unresolved failure
