# F5-T2 — Reissue and Correct an Insufficient Intermediate Result

## Family
Family 5. Orchestration and synthesis

## Goal
Detect that an intermediate result is insufficient, reissue work, and produce a corrected final answer.

## Benchmark intent
This variant tests whether the runtime acts as a real orchestrator rather than a relay layer.

## Expected behavior
- inspect the intermediate output critically
- identify what is missing or weak
- reissue or redirect the task appropriately
- integrate the corrected result into the final answer

## Success criteria
- insufficiency detected correctly
- reissue path is appropriate
- final answer improves materially over the first output
- final report acknowledges the reissue and outcome

## Failure examples
- relaying the weak intermediate output unchanged
- vague critique without corrective action
- incorrect specialist re-routing
- hiding that the first result was insufficient
