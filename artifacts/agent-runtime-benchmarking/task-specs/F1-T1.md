# F1-T1 — New Topic-Home Creation

## Family
Family 1. New topic-home creation

## Goal
Determine that the request is a new topic, create a new topic home under `artifacts/`, and generate an initial durable artifact.

## Benchmark intent
This task measures whether the runtime can correctly apply repository operating rules for new work instead of scattering outputs into unrelated directories.

## Prompt shape
A user asks for a new technical investigation on a topic that does not yet exist in `artifacts/`.

## Expected behavior
- recognize the request as a new topic
- create a new topic home under `artifacts/<topic-home>/`
- generate at least one initial durable artifact (for example `README.md`)
- keep all work inside the new topic home

## Success criteria
- correct topic-home creation
- no writes outside the intended topic home
- initial artifact is coherent and relevant
- final report accurately references the created path and output

## Failure examples
- writing outside the topic home
- modifying an unrelated topic
- answering in chat without creating the durable artifact
- creating the wrong topic-home path

## Data to capture
- created paths
- files written
- tool calls
- wall-clock time
- final report quality
