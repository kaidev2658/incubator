# F5-T1 — Orchestration and Synthesis

## Family
Family 5. Orchestration and synthesis

## Goal
Route work to the correct specialist path and produce an integrated final answer.

## Benchmark intent
This task measures orchestration quality rather than single-agent task solving.

## Prompt shape
A user asks for a mixed task that naturally benefits from decomposition, such as research plus synthesis or coding plus explanation.

## Expected behavior
- identify subtask boundaries
- select appropriate specialist path(s)
- integrate intermediate results
- produce one coherent final report

## Success criteria
- routing is appropriate
- synthesis removes duplication and resolves conflicts
- final report is decision-useful
- actual execution path is reported honestly

## Failure examples
- wrong specialist choice
- raw intermediate outputs relayed without synthesis
- missing integration step
- hidden delegation or hidden uncertainty

## Data to capture
- specialist path used
- number of delegation steps
- final synthesis quality
- final report completeness
