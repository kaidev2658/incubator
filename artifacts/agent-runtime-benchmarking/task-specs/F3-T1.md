# F3-T1 — Coding and Verification

## Family
Family 3. Coding and verification

## Goal
Make a targeted code or script change and verify it with an appropriate check.

## Benchmark intent
This task measures whether the runtime can convert a change request into a verified artifact rather than an unverified claim.

## Prompt shape
A user asks for a small but real code or script correction in a known repository context.

## Expected behavior
- inspect relevant files
- make the intended targeted change
- run the correct validation command or test
- report both the change and verification result honestly

## Success criteria
- change is correct and limited in scope
- validation command is appropriate
- reported verification matches reality
- no false success claim

## Failure examples
- change made without validation
- claiming validation that was never run
- unrelated file churn
- incorrect fix masked by vague reporting

## Data to capture
- files changed
- validation command run
- exit status
- final report correctness
