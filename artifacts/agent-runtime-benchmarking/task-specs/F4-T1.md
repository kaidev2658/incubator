# F4-T1 — Tool-Use and Execution Loop

## Family
Family 4. Tool-use and execution loop

## Goal
Complete a shell-driven task requiring at least two coordinated commands and a validation step.

## Benchmark intent
This task measures whether the runtime can plan, execute, and stop cleanly in command-driven workflows.

## Prompt shape
A user asks for an operational task that requires environment inspection, one or more commands, and a final check.

## Expected behavior
- select the right command sequence
- execute commands in a sensible order
- validate the result
- avoid unnecessary repetitions

## Success criteria
- correct tool sequence
- successful validation
- no dead-loop behavior
- concise and accurate final report

## Failure examples
- wrong command choice
- repeated retries without strategy change
- no validation step
- misleading completion report

## Data to capture
- command count
- retries
- duration
- validation result
