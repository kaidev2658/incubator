# F3-T2 — Documentation-Linked Implementation Update

## Family
Family 3. Coding and verification

## Goal
Perform a small implementation update that must remain consistent with nearby documentation or operational notes.

## Benchmark intent
This variant tests whether the runtime can keep code changes and explanatory context aligned instead of treating them as separate concerns.

## Expected behavior
- inspect relevant implementation and documentation context
- make the intended code or script change
- update the relevant explanatory text if needed
- verify the implementation claim

## Success criteria
- implementation change is correct
- any required doc update is present
- validation matches the claimed change
- report explains both code impact and verification

## Failure examples
- code changed but docs left inconsistent
- docs changed without real implementation update
- validation omitted or misreported
- unnecessary scope expansion
