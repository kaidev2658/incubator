# External vs Internal Benchmarks

External benchmarks provide comparability across systems. They are useful for understanding how an agent runtime performs against widely recognized public tasks such as coding, tool use, browser interaction, or long-horizon reasoning.

Internal benchmarks provide product relevance. They measure whether a runtime can succeed on the workflows that actually matter in the target system, such as topic-home discipline, artifact creation, orchestration quality, verification integrity, and recovery behavior.

For OpenClaw-like systems, external benchmarks are best used as a baseline, while internal workflow benchmarks are needed to evaluate operational fitness.
