# 기술조사 1차 리포트 (KO)

## 1) Wabi

### 제품 정의
- 개인용 소프트웨어 플랫폼 포지션: 미니앱 생성/리믹스/공유
- 자연어 기반 앱 생성 + 리믹스 중심 UX

### 기술 단서
- ToS: AI-powered app generation platform, Mini-App 개념 명시
- 정책 문서 기준 통합/데이터 범위: Gmail, Calendar, Notion, LinkedIn, Reddit, X, Apple Health 등

### 비즈니스/가격
- App Store 표시: 무료(시점 기준)
- 크레딧/구독 구조 단서(ToS) 존재

### 리스크
- 민감 데이터 연동 확대 시 거버넌스/보안 검증 필요
- 런타임 격리/모델별 비용 구조의 공개정보는 제한적

---

## 2) Nothing OS Essential Apps (Essential Space 중심)

### 기능
- 캡처/녹음/노트/링크 등을 모아 기억 보조 허브로 운영
- AI 요약/전사/액션 아이템, 검색/메모리 기능 업데이트 공지
- Calendar 연동, 편집/공유 기능 확장

### UX 차별점
- 입력 마찰 최소화(전용 키+단일 허브)
- 멀티모달 입력을 하나의 타임라인/검색 경험으로 통합

### 리스크
- 온디바이스 vs 클라우드 처리 경계의 명확성 필요
- 권한/프라이버시에 대한 커뮤니티 우려 존재

## 3) 인큐베이터 관점 인사이트
- Wabi형(생성/리믹스/배포) + Nothing형(초저마찰 입력/컨텍스트화) 조합이 PoC에 유효
- 추천 PoC 플로우: 캡처 → 요약/구조화 → 제안 액션 → 승인 실행
