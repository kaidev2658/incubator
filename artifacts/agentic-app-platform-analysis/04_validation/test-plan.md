# Test Plan (PoC v1)

## 1) 목표
Tizen용 Agentic mini-app PoC의 기능/안정성/권한 경계를 빠르게 검증한다.

## 2) 성공 지표 (KPI)
- E2E 성공률: 85% 이상
- 평균 생성+배포 시간: 10초 이내(데모 환경)
- 복구 성공률(롤백): 100%
- 권한 오류 시 안전 실패(fail-safe): 100%

## 3) 테스트 범주
1. 기능 테스트
   - Prompt -> Draft 생성
   - Partial update 반영
   - Rollback 동작
   - 홈 배포/갱신
2. 실패/복구 테스트
   - 네트워크 장애
   - 권한 거부
   - 입력 스키마 오류
3. 보안/권한 테스트
   - 허용 권한 외 접근 차단
   - 사용자 승인 없는 민감 액션 차단

## 4) 대표 E2E 시나리오
### TC-E2E-01
- 입력: "내일 일정+이동 알림 위젯 만들어줘"
- 기대: draft 생성 -> 승인 -> 홈 배포 -> 일정 반영

### TC-E2E-02
- 입력: "주말엔 알림 끄고 평일 오전 8시만"
- 기대: partial update만 적용, 기존 레이아웃 유지

### TC-E2E-03
- 실패 유도: 캘린더 권한 거부
- 기대: 안내 메시지 + 안전 실패 + 재승인 유도

## 5) 산출물
- 테스트 실행 로그
- 실패 케이스 기록(원인/해결)
- KPI 결과표
